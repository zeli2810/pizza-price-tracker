# הרצה יומית מהמחשב שלך (IP ישראלי)

פייס פלוס ווולט חוסמים כתובות IP זרות (שרתי GitHub הם בארה"ב), ולכן הם חייבים
לרוץ מ**מחשב עם IP ישראלי** — המחשב שלך. המדריך הזה מגדיר הרצה אוטומטית יומית.

הרשתות האחרות (דומינוס, פיצה האט, פאפא ג'ונס, פרגו, שמש, סטורי + כמות סניפים)
ממשיכות לרוץ גם בענן (GitHub Actions) — אבל ה-`.bat` מריץ את **הכול**, כך שגם הן
מתעדכנות מהמחשב שלך אם תרצה.

## התקנה חד-פעמית

1. **התקנת ספריות** (מתוך תיקיית `pizza_tracker`):
   ```bat
   pip install -r requirements.txt
   python -m playwright install chromium chrome
   ```

2. **מפתח ה-Service Account**: קח את קובץ ה-JSON שהורדת מ-Firebase (אותו אחד
   שהעלית ל-GitHub), ושמור אותו במקום קבוע מחוץ לפרויקט, למשל:
   ```
   C:\Users\eli\.pizza\serviceAccount.json
   ```
   (צור את התיקייה `.pizza` אם צריך). **אל תשמור אותו בתוך התיקייה של הפרויקט**
   כדי שלא יעלה לגיט בטעות.

3. **ערוך את `run_daily.bat`**: ודא שהשורה
   ```
   set "FIREBASE_SERVICE_ACCOUNT_FILE=C:\Users\eli\.pizza\serviceAccount.json"
   ```
   מצביעה למיקום האמיתי של הקובץ.

4. **בדיקה ידנית**: הרץ פעם אחת בלחיצה כפולה על `run_daily.bat` (או משורת הפקודה).
   בסיום תראה בקובץ `run_daily.log` שהכול רץ, וב-Firestore/דשבורד נתונים טריים.

## תזמון אוטומטי יומי (17:00)

פתח **Command Prompt כמנהל** (Run as administrator) והרץ:
```bat
schtasks /create /tn "PizzaTracker" /tr "C:\_מישהו לחשוב איתו_\הורדות\ai\pizza_tracker\run_daily.bat" /sc daily /st 17:00
```
(עדכן את הנתיב אם הפרויקט נמצא במקום אחר.)

- לבדוק שהמשימה קיימת: `schtasks /query /tn "PizzaTracker"`
- להריץ אותה עכשיו לבדיקה: `schtasks /run /tn "PizzaTracker"`
- למחוק: `schtasks /delete /tn "PizzaTracker" /f`

> המחשב צריך להיות דלוק ומחובר לאינטרנט ב-17:00. אם הוא היה כבוי, אפשר להוסיף
> למשימה הרצה עם הדלקה (בהגדרות Task Scheduler: "Run task as soon as possible
> after a scheduled start is missed").

## הערות
- הרץ **פעם ביום בלבד**. ריצות חוזרות תכופות עלולות לגרום לוולט להחזיר רשימה
  חלקית; יש הגנה שלא נותנת לרשימה חלקית (< 40 מסעדות) לדרוס נתונים טובים.
- וולט מסונן ל**ישראל בלבד** (קישורי `/isr/`) ולאזור ת"א (לפי מיקום המחשב).
- כל סורק שנכשל/נחסם לא דורס את הנתונים הקיימים — פשוט נשמר מה שהיה.
